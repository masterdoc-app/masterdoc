# Эксперимент: ИИ-наполнение ТОиР (фича Technologist)

> Проверено: 2026-07-24 · источники: веб + Telegram MCP + внутр. дизайн · статус: **эксперимент / feature flag**

## Вердикт

Занимаем нишу, которую HubEx/Okdesk/1С не закрывают как продукт: не чат по PDF, а **ИИ, который из документации создаёт draft-структуру ТОиР** (актив → план ППР → чек-лист → preventive WO). Человек только подтверждает. Запускаем как **фичу-эксперимент**, не как обещание «замена FSM».

**С чего начать:** feature `equipment` + demo «1 PDF → пакет draft-карточек» → метрика времени bootstrap.

## One-liner для эксперимента

> Загрузите ИЭ — получите готовый парк и план ТО. ИИ не болтает, а заводит систему.

## Гипотеза эксперимента

| | |
|--|--|
| **H1** | ICP (сеть / сервис 10–50 инженеров) заплатит за пилот, если **время от первого PDF до подтверждённого плана ТО** измеряется часами, а не неделями внедрения 1С/ручного ввода HubEx. |
| **H2** | Чат-ассистент без наполнения системы не конвертит лучше HubEx AI; **bootstrap → Copilot** конвертит. |
| **H3** | Лид уже на HubEx (как АППЕС) купит **надстройку наполнения**, а не миграцию FSM в день 1. |

## Дизайн эксперимента

| Параметр | Значение |
|----------|----------|
| **Формат** | Feature flag `equipment` + закрытый пилот (invite-only org) |
| **Горизонт** | 4–6 недель после готового demo-пути |
| **N пилотов** | 3–5 организаций (ритейл/холод или выездной сервис с PDF на оборудование) |
| **Success** | ≥2 пилота подтвердили ≥5 активов из доков; NPS admin ≥7; повторная загрузка доков без нас |
| **Fail** | Точность draft &lt; порога (много ручных правок) **или** «нам нужен только чат» без подтверждения карточек |
| **Kill criteria** | После 5 пилотов 0 подтверждённых пакетов → откат GTM-клина, оставить Technologist как внутренний tool |

### Метрики

1. **T_bootstrap** — минуты от upload PDF до первого `draft→active` пакета  
2. **Edit ratio** — доля полей, изменённых человеком до confirm  
3. **Activation** — % пилотов с ≥1 confirm за 7 дней  
4. **Pull Copilot** — запросы Copilot на активах, созданных Технологом (вторично)

## Сообщения (эксперимент)

| Канал | Формулировка |
|-------|----------------|
| Demo / sales | «Не ещё один чат. PDF → карточки и ППР на подтверждение.» |
| vs HubEx AI | «У них отвечает на вопросы. У нас заводит систему, потом отвечает.» |
| vs 1С | «Не проект на месяцы — пакет draft за один проход по мануалу.» |
| Не говорить | «Заменим HubEx», «CMMS №1», «AI-first склад» |

## Порядок поставки в эксперименте

```text
1. Technologist  → PDF → draft пакет (ядро эксперимента)
2. Confirm UX    → admin принимает/правит/отклоняет
3. Copilot       → вопрос у того же актива + цитаты
4. (later) Intake / Scribe — не блокеры эксперимента
5. (later) склад / «ждёт запчасть» — отдельный трек боли рынка
```
